## 项目主要需求

**项目概述**：露营基地预约管理系统 —— 面向露营地/房车营地，提供营位可视化预约、装备租赁和活动管理的综合运营平台。

**项目类型**：全栈Web应用

**功能需求**：
1. **营位GIS地图可视化**：以营地平面图展示各营位位置，区分帐篷区、房车区、木屋区，点击营位查看面积、设施（电源/水源/篝火位）和实景照片。
2. **营位类型与预约**：选择日期和入住天数，在地图上点选空闲营位，系统自动计算费用（含营位费+人数附加费），支持连续多日预约。
3. **装备租赁管理**：管理可租赁装备（帐篷/睡袋/炊具/桌椅），记录库存和租赁状态，用户预约营位时可同步选择装备租赁套餐，离营时归还检查。
4. **篝火晚会活动报名**：营地定期举办篝火晚会、星空观测、户外电影等活动，用户可在线报名，查看活动详情（时间/地点/人数限制/费用）。
5. **营位占用日历与设施管理**：以日历视图展示各营位的占用情况，管理员可标记设施维修状态，维修期间该营位不可预约。

**技术要求**：
- 英文短名（用于 Docker Compose）：`ldcampingbase`
- 前端端口：`28509`，后端端口：`29509`
- 前端框架：React 18 + TypeScript
- UI 库：Ant Design
- 构建工具：Vite
- 后端框架：NestJS + TypeScript
- 数据库：MySQL 8.0
- 认证方式：JWT
- 其他依赖：TypeORM、class-validator

后端代码放在 `backend/`：

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=ldcampingbase`，确保 Compose 项目名不依赖目录名（中文目录名会被规范化为空导致 `project name must not be empty` 错误）
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**（Compose V2 已废弃，会产生 `the attribute version is obsolete` 警告）
- 顶层声明 `name: ldcampingbase` 作为兜底，避免目录名为中文时项目名为空
- 所有服务使用 `container_name` 命名容器，便于管理；容器名建议带 `${COMPOSE_PROJECT_NAME:-ldcampingbase}` 前缀
- 数据库数据使用命名卷持久化（`volumes:` 顶层声明，不要使用绑定挂载到中文路径）
- 前端通过 Nginx 反向代理后端 API（前端代码里调用 `/api`，由 Nginx 转发到 `backend:29509`，不要硬编码 `localhost`）
- 服务间通过 Docker 内部网络通信，服务名即主机名
- 数据库服务配置 `healthcheck`，后端服务通过 `depends_on: { db: { condition: service_healthy } }` 等待数据库就绪
- 后端服务也建议配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `80:80`，后端按框架默认端口，数据库使用标准端口
- 所有可变配置（端口、数据库账号密码、JWT 密钥等）通过环境变量注入，并在 `.env.example` 中给出默认值
- 支持通过 `docker compose up -d` 一条命令启动全部服务，不依赖额外的 `-p` 参数

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME`：英文短名（小写字母/数字/`-`/`_`），如 `ldcampingbase`
- 数据库连接相关：`DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_ROOT_PASSWORD`
- 后端：`JWT_SECRET` 或等价的密钥配置
- 端口映射：`FRONTEND_PORT`、`BACKEND_PORT`、`DB_PORT`（可选，便于本地冲突时调整）

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:29509/`
- 配置 `try_files $uri $uri/ /index.html;` 支持前端 SPA 路由
- WebSocket 场景需配置 `Upgrade` / `Connection` 头

自检清单（生成代码后必须确认）：
1. 在 **中文目录名** 下执行 `docker compose config --quiet` 不报错
2. 启动后 `docker compose ps` 显示全部容器 healthy
3. 浏览器访问前端端口能看到页面，且 `/api/...` 请求返回正常
4. README 中的 Docker 启动命令是 `docker compose up -d`（不需要 `-p`）

## docker-compose.yml 参考骨架（务必遵循）

```yaml
name: ldcampingbase   # 顶层 name 字段兜底，避免中文目录名导致 project name 为空

services:
  db:
    image: mysql:8.0
    container_name: ${COMPOSE_PROJECT_NAME:-ldcampingbase}-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${DB_ROOT_PASSWORD}"]
      interval: 5s
      timeout: 5s
      retries: 20

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-ldcampingbase}-backend
    restart: unless-stopped
    environment:
      DB_HOST: db
      DB_PORT: 3306
      DB_NAME: ${DB_NAME}
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-ldcampingbase}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-80}:80"
    depends_on:
      - backend

volumes:
  db_data:
```

```dotenv

```
